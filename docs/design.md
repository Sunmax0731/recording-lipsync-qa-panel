# 設計

## 3 options

1. 汎用メモ型: 実装は軽いが、差別化が弱い。
2. 専用ワークフロー型: 収録・リップシンク検品パネル の課題に集中できる。
3. 大型統合型: 将来性はあるが閉域アルファでは検証範囲が広すぎる。

## Criteria

- 入力負荷を抑えられること。
- 次アクションが迷わず見えること。
- 自動検証と手動テスト手順に落とし込めること。
- プライバシーと配布準備の説明が明確なこと。

## Chosen option

専用ワークフロー型を採用します。理由は、NON PICKUP の優先理由で求められている小型化と差別化を同時に満たせるためです。

## UI direction

- 画面は一覧、詳細、レビュー結果の3領域に分ける。
- ボタン文言は処理結果が分かる動詞にする。
- 8px以下の角丸、安定した寸法、読みやすい日本語文言を守る。
